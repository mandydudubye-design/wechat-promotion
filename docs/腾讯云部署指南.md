
## 📁 第二步：部署项目代码

### 2.1 克隆项目

```bash
# 创建项目目录
mkdir -p /var/www/wechat-promotion
cd /var/www/wechat-promotion

# 克隆代码（替换为你的 Git 仓库地址）
# 方式1：从 GitHub/Gitee 克隆
git clone https://github.com/your-username/wechat-promotion.git .

# 方式2：使用 FTP/SFTP 上传本地代码
# 直接上传整个项目到 /var/www/wechat-promotion
```

### 2.2 配置后端

```bash
cd backend

# 安装依赖
npm install

# 创建生产环境配置
cp .env.example .env.production
vim .env.production
```

配置 `.env.production`：

```env
# 基础配置
NODE_ENV=production
PORT=3000

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=wechat_promotion
DB_USER=root
DB_PASSWORD=你的MySQL密码

# JWT 密钥（生成随机字符串）
JWT_SECRET=生成一个32位以上随机字符串
JWT_EXPIRES_IN=7d

# 微信公众号配置（稍后填写）
WECHAT_APP_ID=你的AppID
WECHAT_APP_SECRET=你的AppSecret
WECHAT_TOKEN=你的Token
WECHAT_ENCODING_AES_KEY=你的EncodingAESKey

# 服务器域名配置
SERVER_DOMAIN=https://x-bussiness.online
```

生成随机密钥：

```bash
# 生成 JWT_SECRET
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 2.3 初始化数据库

```bash
# 导入数据库结构
mysql -u root -p wechat_promotion < init_database.sql

# 或使用 Node.js 脚本初始化
npm run init
```

## 🌐 第三步：配置 Nginx

### 3.1 创建 Nginx 配置文件

```bash
vim /etc/nginx/sites-available/wechat-promotion
```

添加以下配置：

```nginx
# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name x-bussiness.online www.x-bussiness.online;

    # Let's Encrypt 验证路径
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    # 其他请求重定向到 HTTPS
    location / {
        return 301 https://$server_name$request_uri;
    }
}

# HTTPS 配置
server {
    listen 443 ssl http2;
    server_name x-bussiness.online www.x-bussiness.online;

    # SSL 证书路径（稍后配置）
    ssl_certificate /etc/letsencrypt/live/x-bussiness.online/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/x-bussiness.online/privkey.pem;

    # SSL 安全配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # 管理后台（根路径）
    location / {
        root /var/www/wechat-promotion/admin-frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;

        # 缓存静态资源
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # H5 员工端（/h5 路径）
    location /h5 {
        alias /var/www/wechat-promotion/employee-h5/dist;
        try_files $uri $uri/ /h5/index.html;

        # 缓存静态资源
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;

        # WebSocket 支持（如果需要）
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';

        # 代理头
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 超时配置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;

        # 缓存配置（可选）
        proxy_cache_bypass $http_upgrade;
    }

    # 日志
    access_log /var/log/nginx/wechat-promotion-access.log;
    error_log /var/log/nginx/wechat-promotion-error.log;
}
```

### 3.2 启用配置

```bash
# 创建软链接
ln -s /etc/nginx/sites-available/wechat-promotion /etc/nginx/sites-enabled/

# 删除默认配置
rm /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重启 Nginx
systemctl restart nginx
```

## 🔐 第四步：配置 HTTPS 证书

### 4.1 安装 Certbot

```bash
apt install -y certbot python3-certbot-nginx
```

### 4.2 创建验证目录

```bash
mkdir -p /var/www/certbot
```

### 4.3 申请证书

```bash
# 使用 webroot 模式申请
certbot certonly --webroot \
  -w /var/www/certbot \
  -d x-bussiness.online \
  -d www.x-bussiness.online \
  --email your-email@example.com \
  --agree-tos \
  --no-eff-email
```

### 4.4 配置自动续期

```bash
# 测试自动续期
certbot renew --dry-run

# 添加定时任务（每天凌晨2点检查）
crontab -e
```

添加以下行：

```cron
0 2 * * * certbot renew --quiet --post-hook "systemctl reload nginx"
```

## 🔒 第九步：安全加固

### 9.1 配置防火墙

```bash
# 安装 UFW
apt install -y ufw

# 默认拒绝所有入站连接
ufw default deny incoming
ufw default allow outgoing

# 允许 SSH
ufw allow 22/tcp

# 允许 HTTP/HTTPS
ufw allow 80/tcp
ufw allow 443/tcp

# 启用防火墙
ufw enable

# 查看状态
ufw status
```

### 9.2 修改 SSH 端口（可选）

```bash
vim /etc/ssh/sshd_config
```

修改：

```
Port 2222  # 改为非默认端口
PermitRootLogin no  # 禁止 root 直接登录
```

重启 SSH：

```bash
systemctl restart sshd
```

### 9.3 限制 MySQL 远程访问

```bash
vim /etc/mysql/mysql.conf.d/mysqld.cnf
```

确保：

```
bind-address = 127.0.0.1
```

重启 MySQL：

```bash
systemctl restart mysql
```

```bash
#!/bin/bash

    echo "后端服务异常，已尝试重启" | mail -s "【告警】后端服务异常" your-email@example.com

    echo "Nginx 服务异常，已尝试重启" | mail -s "【告警】Nginx 服务异常" your-email@example.com
    systemctl restart nginx
fi

# 检查 MySQL
if ! systemctl is-active --quiet mysql; then
    echo "MySQL 服务异常，已尝试重启" | mail -s "【告警】MySQL 服务异常" your-email@example.com
    systemctl restart mysql
fi
EOF

chmod +x /var/www/wechat-promotion/check-services.sh

# 添加定时任务（每10分钟检查一次）
crontab -e
```

添加：

## 📞 常见问题

### 1. HTTPS 证书申请失败

**原因**: DNS 解析未生效或 Nginx 配置错误

**解决**:
```bash
# 检查 DNS 解析
nslookup x-bussiness.online

# 检查 Nginx 配置
nginx -t

# 重新申请证书
certbot certonly --webroot -w /var/www/certbot -d x-bussiness.online
```

### 2. 微信验证失败

**原因**: Token 不匹配或服务器无法访问

**解决**:
```bash
# 检查后端日志
pm2 logs wechat-backend

# 确认 .env.production 中的 WECHAT_TOKEN 与微信后台一致
cat /var/www/wechat-promotion/backend/.env.production | grep WECHAT_TOKEN

# 测试 webhook 可访问性
curl https://x-bussiness.online/api/wechat/webhook
```

### 3. 前端无法访问后端

**原因**: CORS 或代理配置错误

**解决**:
```bash
# 检查后端 CORS 配置
# 应该允许所有域名或指定 x-bussiness.online

# 检查 Nginx 代理配置
nginx -t
systemctl reload nginx
```

### 4. 数据库连接失败

**原因**: 密码错误或 MySQL 未启动

**解决**:
```bash
# 检查 MySQL 状态
systemctl status mysql

# 测试连接
mysql -u root -p -h localhost wechat_promotion

# 检查 .env.production 配置
cat /var/www/wechat-promotion/backend/.env.production | grep DB_
```
