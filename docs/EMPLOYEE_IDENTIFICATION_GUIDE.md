# 员工身份识别系统 - 原理说明

## 🤔 你的问题

> "关注员工，是不是得把员工的微信昵称收集回来才能识别到公众号后台的用户哪些是现在的员工"

## 📊 两种不同的识别需求

### 需求1：推广归属识别（已实现✅）
**问题**：这个用户是哪个员工推广来的？

**答案**：不需要知道用户真实身份，通过二维码的 `scene_str` 即可识别

**流程**：
```
用户扫码（带场景值：scene_emp001）
    ↓
微信推送关注事件（包含 scene_emp001）
    ↓
后端解析 scene_str，知道这是员工 EMP001 推广的
    ↓
保存关注记录：{ employee_id: "EMP001", openid: "oxxx", nickname: "张三" }
    ↓
✅ 完成推广归属识别
```

**关键点**：
- ✅ 通过二维码的 `scene_str`（如 `scene_emp001_001`）识别
- ✅ 不需要知道用户真实身份
- ✅ 只需要记录"这个用户是EMP001推广的"

### 需求2：员工身份识别（需要新功能🆕）
**问题**：这个关注用户是我们公司的员工吗？

**答案**：需要验证用户身份，将微信账号与员工关联

**流程**：
```
用户关注公众号
    ↓
保存关注记录（不知道是否是员工）
    ↓
用户在H5页面验证身份（输入手机号+验证码）
    ↓
验证手机号是否在员工表中
    ↓
如果是，绑定关系：openid ↔ employee_id
    ↓
✅ 完成员工身份识别
```

## 🎯 实际应用场景

### 场景A：对外推广（不需要识别员工身份）

**公司场景**：销售公司，员工推广产品

**关注用户**：都是外部客户

**识别需求**：
- ❌ 不需要知道用户是否是员工（因为都是客户）
- ✅ 只需要知道是哪个员工推广的

**系统功能**：
```
EMP001（张三）推广 → 客户A扫码 → 记录：客户A是EMP001推广的
EMP002（李四）推广 → 客户B扫码 → 记录：客户B是EMP002推广的
```

**统计结果**：
- EMP001 推广了 100 个客户
- EMP002 推广了 80 个客户

### 场景B：内部公众号（需要识别员工身份）

**公司场景**：公司内部公众号，发布通知、新闻

**关注用户**：公司员工 + 外部人员

**识别需求**：
- ✅ 需要知道哪些关注者是员工
- ✅ 统计员工关注率
- ✅ 推广归属不重要（都是公司自己的号）

**系统功能**：
```
用户A关注 → 验证手机号 → 是员工！绑定 employee_id
用户B关注 → 验证手机号 → 不是员工 → 保持普通用户
用户C关注 → 未验证 → 身份未知
```

**统计结果**：
- 公司员工总数：100人
- 已关注员工：80人
- 员工关注率：80%

### 场景C：混合场景（两者都需要）

**公司场景**：员工内部推广，同时追踪推广效果

**关注用户**：员工（互相推广）+ 外部客户

**识别需求**：
- ✅ 需要区分员工和客户
- ✅ 需要知道是哪个员工推广的

**系统功能**：
```
EMP001 推广的二维码
    ↓
用户A扫码 → 是员工 EMP002 → 记录：员工互推
    ↓
用户B扫码 → 不是员工 → 记录：EMP001 的推广成果
```

## 🔍 如何识别员工

### 方法1：自动识别（企业微信）

如果是企业微信公众号，可以通过 `unionid` 自动识别：

```typescript
// 数据库添加字段
ALTER TABLE employees ADD COLUMN wechat_unionid VARCHAR(100);

// 自动识别逻辑
用户关注 → 获取 unionid → 在员工表中查找 → 找到则自动绑定
```

### 方法2：手机号验证（推荐）

让用户主动验证身份：

```typescript
// 1. 用户在H5页面输入手机号
// 2. 系统发送验证码
// 3. 用户输入验证码
// 4. 验证手机号是否在员工表中
// 5. 如果是，绑定关系
```

### 方法3：管理员手动识别

管理员查看关注列表，根据昵称手动识别：

```typescript
// 1. 管理员查看关注记录
// 2. 看到昵称"张三"，手机号"138..." 
// 3. 在员工表中搜索
// 4. 找到对应员工，手动绑定
```

## 📋 数据库设计

### 推广归属表（已存在）

```sql
promotion_records:
- employee_id: 员工ID（谁推广的）
- scene_str: 场景值（识别用）
- scan_count: 扫码数
- follow_count: 关注数
```

### 关注记录表（已存在，需添加字段）

```sql
follow_records:
- openid: 微信OpenID
- employee_id: 推广员工ID（谁推广的）
- nickname: 微信昵称
- is_employee: 是否是员工 🆕
- employee_binding_id: 员工绑定ID 🆕
```

### 员工绑定表（新增）

```sql
employee_bindings:
- employee_id: 员工ID
- openid: 微信OpenID
- unionid: 微信UnionID（企业微信）
- is_verified: 是否已验证
- verification_method: 验证方式
```
