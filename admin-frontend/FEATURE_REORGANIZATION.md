# 功能重新划分说明

## 📋 问题说明

用户反馈：**管理后台的"公众号管理"和"系统设置"都有公众号AppID和Secret管理，功能重复且矛盾。**

---

## ✅ 解决方案

### 功能重新划分

#### 📱 公众号管理页面（`/accounts`）
**职责**：管理多个微信公众号的业务配置

**功能**：
- ✅ 添加/编辑/删除公众号
- ✅ 公众号基本信息（名称、描述、头像）
- ✅ 公众号状态管理（启用/停用）
- ✅ 查看公众号统计（推广员工数、扫码数等）
- ✅ 公众号列表展示

**包含的信息**：
```typescript
{
  id: string;
  accountName: string;        // 公众号名称
  appId: string;              // AppID（只读）
  description: string;        // 描述
  avatar: string;             // 头像
  status: 'active' | 'inactive';  // 状态
  createdAt: string;          // 创建时间
}
```

**不包含**：
- ❌ AppSecret（安全考虑，不在列表中显示）
- ❌ 详细API配置（Token、EncodingAESKey等）

---

#### ⚙️ 系统设置页面（`/settings`）
**职责**：配置系统级参数、安全和数据同步

**功能模块**：

1. **系统信息**
   - 系统版本
   - 可用性状态
   - 数据库状态
   - API状态

2. **数据同步**
   - 自动同步间隔
   - 上次同步时间
   - 手动同步按钮

3. **通知设置**
   - 新关注通知开关
   - 每日汇总开关
   - 异常提醒开关
   - 通知邮箱配置

4. **推广设置**
   - 推广二维码有效期
   - 排行榜展示周期
   - 每日推广任务目标

5. **安全设置**
   - 强制绑定验证
   - IP白名单
   - 会话超时时间
   - 密码复杂度要求

6. **员工管理设置**
   - 员工ID前缀
   - 默认员工角色
   - 新员工试用期
   - 每月最低推广要求

**不包含**：
- ❌ 公众号AppID/Secret配置（已移到公众号管理）
- ❌ 公众号列表展示
- ❌ 公众号状态管理

---

## 💡 设计理念

### 1. 职责分离原则
- **公众号管理**：专注于公众号的业务管理
- **系统设置**：专注于系统的全局配置

### 2. 安全性考虑
- AppSecret不在列表中显示（防止泄露）
- 如需修改API参数，在公众号详情中配置

### 3. 用户体验
- 运营人员关注公众号管理
- 管理员关注系统设置
- 功能清晰，不会混淆

### 4. 扩展性
- 公众号管理可以添加更多公众号相关功能
- 系统设置可以添加更多系统级配置

## 📝 使用指南

### 场景1：添加新公众号
1. 访问 **公众号管理**（`/accounts`）
2. 点击"添加公众号"
3. 填写公众号信息
4. 保存

### 场景2：配置系统参数
1. 访问 **系统设置**（`/settings`）
2. 找到对应的功能模块
3. 修改配置
4. 保存设置

### 场景3：启用/停用公众号
1. 访问 **公众号管理**（`/accounts`）
2. 找到目标公众号
3. 点击"启用"或"停用"按钮

### 场景4：修改同步间隔
1. 访问 **系统设置**（`/settings`）
2. 找到"数据同步"卡片
3. 选择同步间隔
4. 保存设置

## 🎯 总结

### 修改前的问题
- ❌ 功能重复（两个页面都有公众号配置）
- ❌ 职责不清（不知道在哪里配置什么）
- ❌ 用户体验差（容易混淆）

### 修改后的优势
- ✅ 职责清晰（公众号管理 vs 系统设置）
- ✅ 功能独立（各自管理自己的配置）
- ✅ 用户友好（明确的功能划分）
- ✅ 易于扩展（可以独立添加功能）
