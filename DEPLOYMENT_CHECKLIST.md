# 部署检查清单

## 🔍 快速检查步骤

### 1. 在 Railway 控制台获取信息

访问：https://railway.app/dashboard

检查：
- [ ] 项目状态显示 "Running"（绿色）
- [ ] 复制你的应用域名（Domain）
- [ ] 检查是否有错误提示

### 2. 测试前端访问

在浏览器打开你的域名：

- [ ] 页面能正常加载
- [ ] 看到登录界面
- [ ] 没有控制台错误（按 F12 查看）

### 3. 测试 API

在浏览器访问：`https://your-domain/api/health`

- [ ] 返回 JSON 数据
- [ ] 状态码是 200

### 4. 测试登录

使用默认账户：
- 用户名：`admin`
- 密码：`admin123`

- [ ] 登录成功
- [ ] 进入系统主页
- [ ] 能看到功能菜单

### 5. 修改密码

登录后：
- [ ] 找到修改密码功能
- [ ] 成功修改默认密码
- [ ] 退出后能用新密码登录

## ❓ 如果检查失败

### 问题 1：页面打不开（404/500）

**可能原因：**
- 环境变量未配置
- 后端服务启动失败
- 前端文件未正确复制

**解决方法：**
1. 检查 Railway 控制台的日志
2. 确认环境变量已配置
3. 查看部署状态

### 问题 2：API 返回错误

**可能原因：**
- 数据库未初始化
- 数据库连接失败
- JWT_SECRET 未配置

**解决方法：**
1. 运行初始化脚本：`railway run "npm run init-admin"`
2. 检查数据库环境变量
3. 查看日志排查错误

### 问题 3：无法登录

**可能原因：**
- 数据库未初始化
- 管理员账户未创建

**解决方法：**
1. 运行：`railway run "npm run init-admin"`
2. 检查数据库是否正常
3. 查看 API 日志

## 📊 查看日志

### 在命令行查看：
```bash
cd C:\公众号任务
railway logs
```

### 在控制台查看：
1. 打开 Railway 控制台
2. 选择你的项目
3. 点击 "Logs" 标签
4. 查看实时日志
