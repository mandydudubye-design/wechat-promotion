# 订阅号/服务号双模式功能测试指南

## 📋 功能概述

本系统现已支持**订阅号**和**服务号**两种公众号类型，根据公众号类型自动选择不同的员工绑定方式：
- **服务号**：H5网页授权（静默授权，用户无感知）
- **订阅号**：识别码验证（员工在公众号对话框发送识别码）

---

## 🚀 快速开始

### 1. 启动后端服务

```bash
cd c:/公众号任务/backend
npm run dev
```

后端将运行在 `http://localhost:3000`

### 2. 启动前端服务（新开一个终端）

```bash
cd c:/公众号任务/frontend
npm run dev
```

前端将运行在 `http://localhost:5173`

### 3. 执行数据库迁移

**方式一：通过 API 执行（推荐）**

访问：`http://localhost:3000/api/init/migrate-account-type`

或者使用 curl：
```bash
curl -X POST http://localhost:3000/api/init/migrate-account-type
```

**方式二：手动执行 SQL**

```bash
cd c:/公众号任务/backend
node src/scripts/migrate_account_type.js
```

## 📝 测试流程

### 步骤 1：登录系统

1. 打开浏览器访问：`http://localhost:5173`
2. 使用管理员账号登录：
   - 用户名：`admin`
   - 密码：`admin123`

### 步骤 2：添加公众号

1. 进入"多公众号管理"页面
2. 点击"添加公众号"按钮
3. 填写公众号信息：
   - **公众号名称**：例如"测试服务号"
   - **公众号ID**：例如"gh_test123"
   - **公众号类型**：选择"服务号"或"订阅号"
   - **AppID**：（可选）填入你的微信公众号 AppID
   - **AppSecret**：（可选）填入你的微信公众号 AppSecret
4. 点击"添加"

### 步骤 3：添加员工

1. 进入"员工管理"页面
2. 点击"添加员工"按钮
3. 填写员工信息：
   - **工号**：例如"EMP001"
   - **姓名**：例如"张三"
   - **部门**：例如"技术部"
   - **职位**：例如"开发工程师"
   - **手机号**：例如"13800138000"
4. 点击"添加"

### 步骤 4：测试员工绑定

#### 服务号绑定流程

1. 进入"员工管理"页面，找到刚才添加的员工
2. 点击"绑定微信"按钮
3. 系统会自动跳转到微信授权页面
4. 在微信内完成授权（静默授权，用户无感知）
5. 输入工号和手机号后4位验证
6. 绑定成功

**测试链接**：`http://your-domain.com/employee-h5/employee-bind?account_id=1`

#### 订阅号绑定流程

1. 在微信公众号对话框中发送识别码：
   ```
   EMP0010001
   ```
   （格式：工号 + 手机号后4位）

2. 系统会回复："验证成功！请打开绑定页面完成绑定"

3. 打开员工绑定页面：
   ```
   http://your-domain.com/employee-h5/employee-bind?account_id=1
   ```

4. 输入工号和手机号后4位验证
5. 绑定成功

## 🔍 验证功能

### 1. 检查公众号类型

访问：`http://localhost:3000/api/accounts`

查看返回的 `account_type` 字段是否正确：
- `subscription`：订阅号
- `service`：服务号

### 2. 检查绑定配置

访问：`http://localhost:3000/api/employee-unified/bind-config?account_id=1`

查看返回的配置信息：
```json
{
  "success": true,
  "data": {
    "account_id": 1,
    "account_name": "测试服务号",
    "account_type": "service",
    "bind_method": "oauth",
    "supports_oauth": true,
    "supports_code": false
  }
}
```

### 3. 检查员工绑定状态

进入"员工管理"页面，查看员工的"绑定状态"列：
- **未绑定**：未绑定微信
- **已绑定**：已成功绑定微信

## 📊 API 端点说明

### 新增端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/init/migrate-account-type` | POST | 执行数据库迁移（添加公众号类型字段） |
| `/api/employee-unified/bind-config` | GET | 获取公众号绑定配置 |
| `/api/employee-unified/bind` | POST | 统一员工绑定接口 |
| `/api/verification-code/send` | POST | 发送识别码（订阅号） |
| `/api/verification-code/verify` | POST | 验证识别码（订阅号） |
| `/api/wechat-callback/unified` | POST | 微信统一回调处理 |
| `/api/wechat-oauth/auth-url` | GET | 获取微信授权 URL（服务号） |
| `/api/wechat-oauth/callback` | GET | 处理微信授权回调（服务号） |

## 🐛 常见问题

### 1. 数据库迁移失败

**问题**：执行迁移时提示"字段已存在"

**解决**：这是正常的，表示字段已经存在，可以忽略。

### 2. 微信授权失败

**问题**：服务号绑定时提示"授权失败"

**解决**：
- 检查 AppID 和 AppSecret 是否正确
- 检查网页授权域名是否配置（微信公众平台 → 开发 → 接口权限 → 网页授权）
- 必须在微信内打开链接（浏览器打开会失败）

### 3. 识别码验证失败

**问题**：订阅号绑定时提示"识别码错误"

**解决**：
- 检查识别码格式是否正确（工号 + 手机号后4位）
- 检查员工信息是否正确
- 确保识别码是在公众号对话框中发送的

### 4. 后端启动失败

**问题**：提示"数据库连接失败"

**解决**：
- 检查 MySQL 服务是否启动
- 检查 `.env` 文件中的数据库配置是否正确
- 确认数据库 `wechat_promotion` 是否存在
