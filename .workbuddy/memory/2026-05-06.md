**解决方案**：实现 H5 落地页模式，不依赖微信 API，适用于所有类型公众号

**实现内容**：

   - `employee-h5/src/pages/PromotionLandingPage.tsx` - 落地页面组件
   - `employee-h5/src/pages/PromotionLandingPage.css` - 落地页样式

   - 修改 `backend/src/services/promotionService.ts`，统一使用 H5 落地页模式
   - 生成 H5 落地页 URL：`{baseUrl}/landing?accountId={accountId}&employeeId={employeeId}`
   - 使用第三方 API 生成落地页二维码

3. **添加扫码统计 API**：
   - 创建数据库表 `scan_records` - 记录每次扫码行为

4. **更新 H5 首页**：
   - 添加 `generatePromotionCode()` 函数调用后端 API

**功能特点**：
- ✅ 扫码后打开精美的 H5 落地页
- ✅ 显示公众号二维码和关注引导
- ✅ 自动记录扫码统计数据
- ✅ 适用于所有类型的公众号（服务号/订阅号/认证/未认证）
- ✅ 不需要配置微信 API 和 IP 白名单

## 修复后端启动卡住问题

**根因**：`promotion-old.ts` 旧文件有 TS 编译错误（`verificationCode` 属性不存在），导致 `tsc` 编译失败。而 `npm run dev` 用 `ts-node` 跳过了类型检查所以看起来能启动，但后台启动（编译后执行）会失败。

**修复**：删除 `promotion-old.ts`，同时优化 `app.ts` 中 `promotion_kits` 表的 ALTER TABLE 逻辑，先查字段是否存在再操作。

## 修复二维码预览 + 扫码白屏

**问题1：管理后台上传二维码预览不出来**
- 根因：`app.ts` 用 `__dirname`（指向 `dist/public`）做静态文件服务，而 `upload.ts` 用 `process.cwd()`（指向 `backend/public`）存上传文件，路径不一致
- 修复：`app.ts` 统一改为 `process.cwd()`，并增加 `/uploads` 专用静态路由

**问题2：H5 扫码进入落地页白屏**
- 根因：之前修改代码时遗漏了 `isVerified` 变量声明，第 133 行引用了未定义变量，React 渲染崩溃
- 修复：补回 `const isVerified = account.verified === true || account.verified === 1;`

## 修复公众号二维码不显示

**根因**：管理后台上传二维码时用了 `URL.createObjectURL()` 生成 blob URL 直接存入数据库，blob URL 只在浏览器内存中有效，手机扫码后无法访问。

**修复**：
1. `AccountsPage.tsx`：上传二维码改为先调后端 `/api/upload/image` 接口，获取真实的 `/uploads/xxx.png` 路径
2. `PromotionLandingPage.tsx`：过滤 blob URL，将相对路径拼接为完整 URL
3. 已有数据库中的 blob URL 数据会回退显示微信号让用户手动搜索

**注意**：需要在管理后台重新上传公众号二维码图片，才能在落地页正常显示。

## 修复扫码后网络错误

**根因**：二维码 URL 使用 `localhost:5174`，手机扫码后无法访问电脑的本地地址

**修复**：
1. 后端 `promotionService.ts`：开发环境 URL 从 `localhost:5174` 改为 `192.168.100.200:5174`
2. 落地页 `PromotionLandingPage.tsx`：所有 `localhost:3000` API 调用改为局域网 IP 常量 `API_BASE`
3. 支持 `H5_BASE_URL` 环境变量覆盖，生产环境仍使用 `SERVER_URL`

**关键配置**：
- Vite 已配置 `host: '0.0.0.0'` 允许局域网访问
- 开发环境局域网 IP：`192.168.100.200`
- 手机和电脑需在同一 WiFi 下

## 修复推广码初始化失败问题

**根因**：双重 bug 导致进入H5就报错
1. **React state 异步问题**：useEffect 里先用 mock 数据设置 `selectedAccount`（含无效 id），立即触发生成推广码，用 mock id 查数据库找不到记录
2. **id 类型不匹配**：后端 `/api/employee-binding/accounts` 返回的 `id` 是字符串，传给生成推广码 API 时需要整数

**修复方案**：
- 删除 mock 数据初始化 selectedAccount 的逻辑，改为全由后端数据驱动
- 在 `fetchAccounts` 中直接传入数据生成推广码，不等 React state 异步更新
- 前端 `generatePromotionCode` 加 `parseInt` 转换 accountId
- 后端 `smart-promotion.ts` 路由也加 `parseInt` 保险
