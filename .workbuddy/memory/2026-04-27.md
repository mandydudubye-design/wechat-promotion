## 双账号类型功能实现

### 背景
原有数据库架构中 `wechat_accounts` 表存储的是微信用户/粉丝信息，缺少公众号配置管理。需要支持订阅号和服务号两种类型的账号配置。

### 实施内容

#### 1. 数据库层面
- 创建新表 `wechat_account_configs` 存储公众号配置信息
- 字段包括：account_name, account_id, app_id, app_secret, account_type (subscription/service), qr_code_url, status
- 插入测试数据：测试订阅号、测试服务号

#### 2. 后端服务
- 新增 `wechatAccountConfigService.ts`：提供公众号配置的增删改查服务
- 新增 `wechat-account-configs.ts` 路由：提供 RESTful API
  - GET /api/wechat-configs - 获取所有配置
  - GET /api/wechat-configs/:appId - 按 AppID 查询
  - GET /api/wechat-configs/type/:type - 按类型（subscription/service）查询
  - POST /api/wechat-configs - 创建/更新配置
  - DELETE /api/wechat-configs/:appId - 删除配置
- 在 `app.ts` 中注册新路由

#### 3. 验证
- 编译成功，后端服务启动正常
- API 测试通过：
  - 健康检查：http://localhost:3000/health
  - 配置列表：http://localhost:3000/api/wechat-configs
  - 按类型查询：http://localhost:3000/api/wechat-configs/type/service

### 后台登录问题修复
**问题**：登录时报错 `Table 'wechat_promotion.admins' doesn't exist`

**原因**：
- 数据库中缺少 `admins` 表
- 后端认证逻辑依赖 `admins` 表进行用户验证

**解决方案**：
1. 创建 `admins` 表，包含字段：id, username, password, name, email, role, created_at, updated_at
2. 运行 `src/scripts/initAdmin.ts` 脚本初始化默认管理员账号
3. 默认登录凭证：
   - 用户名：`admin`
   - 密码：`admin123`

### 白屏问题排查与修复
**问题**：登录后点击任何功能都会白屏，报错 `TypeError: Cannot read properties of undefined (reading 'length')`

**根本原因**：
- 多个页面组件在数据未加载完成时就尝试访问数组的 `.map()` 和 `.length` 属性
- API 返回的数据可能暂时是 `undefined`，导致空值访问错误

**修复措施**：
1. 创建 `ErrorBoundary` 组件捕获并显示详细错误信息
2. 修改 `Dashboard.tsx`：使用安全的默认值，避免依赖 mock 数据
3. 修改 `EmployeesPage.tsx`：添加空值保护 `employees || []`
4. 修改 `FollowStatusPage.tsx`：添加空值保护
5. 修改 `ReportsPage.tsx`：添加空值保护

**具体修复代码**：
```typescript
// 修复前
const departments = ['全部部门', ...new Set(employees.map(e => e.department).filter(Boolean) as string[])]

// 修复后
const departments = ['全部部门', ...new Set(
  (employees || [])
    .map(e => e.department)
    .filter((dept): dept is string => Boolean(dept))
)]
```

**状态**：✅ 已修复所有页面的空值访问问题
