# 公众号推广追踪系统 PRD

## 一、产品概述

### 1.1 产品背景
某国企作为行政负责人需要管理组织员工关注官方公众号的情况，并通过员工转发推广来扩大公众号影响力。本系统旨在提供一套完整的员工关注管理、推广追踪及效果统计解决方案。

### 1.2 产品目标
- 自动识别组织员工对公众号的关注状态
- 为员工生成专属推广二维码，追踪推广效果
- 统计员工转发带来的新增关注数据
- 提供数据报表支持管理决策

### 1.3 目标用户
- **主要用户**：国企行政负责人/管理员
- **次要用户**：组织内部员工

### 3.2 M01 - 员工管理模块

#### 3.2.1 功能概述
实现员工信息的录入、审核、绑定等全生命周期管理。

#### 3.2.2 功能列表

| 功能编号 | 功能名称 | 优先级 | 说明 |
|---------|---------|-------|------|
| M01-01 | 员工信息导入 | P0 | 批量导入员工基本信息 |
| M01-02 | 员工绑定 | P0 | 员工微信账号与系统账号绑定 |
| M01-03 | 员工列表 | P0 | 查看、搜索、筛选员工 |
| M01-04 | 员工详情 | P1 | 查看员工详细信息 |
| M01-05 | 员工编辑 | P1 | 修改员工信息 |
| M01-06 | 员工删除/禁用 | P2 | 删除或禁用员工账号 |

#### 3.2.3 详细功能说明

**M01-01 员工信息导入**
- **入口**：管理后台 - 员工管理 - 导入员工
- **功能描述**：管理员通过Excel模板批量导入员工信息
- **字段要求**：
  - 工号（必填，唯一）
  - 姓名（必填）
  - 部门（必填）
  - 职位（选填）
  - 手机号（选填，用于验证）
  - 邮箱（选填）
- **校验规则**：
  - 工号不能重复
  - 手机号格式校验
  - 必填字段校验
- **交互流程**：
  1. 下载导入模板
  2. 填写员工信息
  3. 上传文件
  4. 系统校验并显示导入结果
  5. 确认导入

**M01-02 员工绑定**
- **入口**：员工扫码进入H5页面
- **功能描述**：员工通过微信扫码完成账号绑定
- **绑定方式**：
  - 方式一：工号+姓名验证
  - 方式二：工号+手机号验证
- **交互流程**：
  1. 员工扫描绑定二维码
  2. 进入绑定页面
  3. 输入工号、姓名/手机号
  4. 系统验证信息
  5. 验证成功，获取用户OpenID
  6. 完成绑定，生成推广码
- **绑定状态**：
  - 未绑定：员工已录入但未完成微信绑定
  - 已绑定：员工完成微信绑定
  - 已禁用：员工账号被禁用

**M01-03 员工列表**
- **入口**：管理后台 - 员工管理
- **展示字段**：
  - 工号
  - 姓名
  - 部门
  - 绑定状态
  - 关注状态
  - 推广人数
  - 绑定时间
- **支持操作**：
  - 搜索：按工号、姓名、部门搜索
  - 筛选：按绑定状态、关注状态、部门筛选
  - 排序：按工号、绑定时间、推广人数排序
  - 导出：导出员工列表Excel

#### 3.2.4 数据模型

```sql
-- 员工表
CREATE TABLE employees (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  employee_no VARCHAR(50) UNIQUE NOT NULL COMMENT '工号',
  name VARCHAR(100) NOT NULL COMMENT '姓名',
  department VARCHAR(100) NOT NULL COMMENT '部门',
  position VARCHAR(100) COMMENT '职位',
  phone VARCHAR(20) COMMENT '手机号',
  email VARCHAR(100) COMMENT '邮箱',
  openid VARCHAR(100) UNIQUE COMMENT '微信OpenID',
  unionid VARCHAR(100) COMMENT '微信UnionID',
  bind_status TINYINT DEFAULT 0 COMMENT '绑定状态 0-未绑定 1-已绑定 2-已禁用',
  bind_time DATETIME COMMENT '绑定时间',
  follow_status TINYINT DEFAULT 0 COMMENT '关注状态 0-未关注 1-已关注',
  follow_time DATETIME COMMENT '关注时间',
  unfollow_time DATETIME COMMENT '取消关注时间',
  qr_code_url VARCHAR(255) COMMENT '推广二维码URL',
  scene_id VARCHAR(100) UNIQUE COMMENT '场景ID',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_employee_no (employee_no),
  INDEX idx_openid (openid),
  INDEX idx_department (department)
) COMMENT '员工表';
```

### 3.3 M02 - 关注状态查询模块

#### 3.3.1 功能概述
实时同步和查询员工对公众号的关注状态。

#### 3.3.2 功能列表

| 功能编号 | 功能名称 | 优先级 | 说明 |
|---------|---------|-------|------|
| M02-01 | 关注状态同步 | P0 | 从微信同步关注者列表 |
| M02-02 | 关注状态查询 | P0 | 查看员工关注状态 |
| M02-03 | 关注变更通知 | P1 | 关注/取关事件实时处理 |

#### 3.3.3 详细功能说明

**M02-01 关注状态同步**
- **同步方式**：
  - 定时同步：每小时自动同步一次
  - 手动同步：管理员点击同步按钮
  - 事件触发：微信推送关注/取关事件时实时更新
- **同步逻辑**：
  1. 调用微信API获取关注者OpenID列表
  2. 遍历员工表，匹配OpenID
  3. 更新员工关注状态
  4. 记录同步日志
- **性能优化**：
  - 增量同步：只同步最近变更的用户
  - 异步处理：大批量同步使用队列

**M02-02 关注状态查询**
- **展示方式**：
  - 员工列表中的状态列
  - 专门的关注状态统计页面
- **状态说明**：
  - 已关注：员工当前关注公众号
  - 未关注：员工当前未关注公众号
  - 未绑定：员工未完成账号绑定

**M02-03 关注变更通知**
- **触发时机**：员工关注/取消关注公众号
- **通知方式**：
  - 系统内消息
  - 可选：邮件/短信通知管理员
- **记录内容**：
  - 变更时间
  - 变更类型（关注/取关）
  - 员工信息

#### 3.3.4 数据模型

```sql
-- 关注记录表
CREATE TABLE follow_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  employee_id BIGINT NOT NULL COMMENT '员工ID',
  openid VARCHAR(100) NOT NULL COMMENT '微信OpenID',
  event_type TINYINT NOT NULL COMMENT '事件类型 1-关注 2-取关',
  event_time DATETIME NOT NULL COMMENT '事件时间',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_employee_id (employee_id),
  INDEX idx_openid (openid),
  INDEX idx_event_time (event_time),
  FOREIGN KEY (employee_id) REFERENCES employees(id)
) COMMENT '关注记录表';
```

### 3.4 M03 - 推广二维码管理模块

#### 3.4.1 功能概述
为每个员工生成专属推广二维码，管理二维码的生成、下载、统计。

#### 3.4.2 功能列表

| 功能编号 | 功能名称 | 优先级 | 说明 |
|---------|---------|-------|------|
| M03-01 | 生成推广码 | P0 | 为员工生成专属推广二维码 |
| M03-02 | 推广码下载 | P0 | 下载推广二维码图片 |
| M03-03 | 推广码管理 | P1 | 查看、重置、禁用推广码 |
| M03-04 | 推广码分享 | P0 | 员工端分享推广码 |

#### 3.4.3 详细功能说明

**M03-01 生成推广码**
- **生成时机**：
  - 员工完成绑定时自动生成
  - 管理员手动为员工生成
  - 重置推广码时重新生成
- **生成规则**：
  - 使用微信带参数二维码接口
  - 场景值 = 员工ID（加密处理）
  - 二维码类型：永久二维码
- **二维码样式**：
  - 默认样式：公众号头像 + 公众号名称
  - 可选：自定义样式（添加推广语、员工姓名等）

**M03-02 推广码下载**
- **下载格式**：PNG图片
- **下载方式**：
  - 管理后台批量下载
  - 员工H5页面单独下载
- **文件命名**：`推广码_姓名_工号.png`

**M03-03 推广码管理**
- **查看推广码**：
  - 员工列表中直接预览
  - 点击查看大图
- **重置推广码**：
  - 重新生成新的推广码
  - 原推广码失效
  - 记录重置日志
- **禁用推广码**：
  - 临时禁用员工推广功能
  - 禁用期间扫码不记录推荐关系

**M03-04 推广码分享**
- **员工端入口**：H5页面 - 我的推广码
- **分享方式**：
  - 保存图片到相册
  - 直接分享给微信好友
  - 生成分享海报
- **分享内容**：
  - 推广二维码
  - 公众号名称
  - 推广语（可自定义）

#### 3.4.4 数据模型

```sql
-- 推广记录表
CREATE TABLE promotion_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  employee_id BIGINT NOT NULL COMMENT '员工ID',

  new_user_nickname VARCHAR(100) COMMENT '新用户昵称',
  scan_time DATETIME NOT NULL COMMENT '扫码时间',
  subscribe_time DATETIME COMMENT '关注时间',

  INDEX idx_scene_id (scene_id),
  INDEX idx_new_user_openid (new_user_openid),

  FOREIGN KEY (employee_id) REFERENCES employees(id)
) COMMENT '推广记录表';
```

### 3.5 M04 - 推广效果统计模块

#### 3.5.1 功能概述
统计员工推广数据，包括扫码量、关注量、推广排名等。

#### 3.5.2 功能列表

| 功能编号 | 功能名称 | 优先级 | 说明 |
|---------|---------|-------|------|
| M04-01 | 推广数据统计 | P0 | 统计员工推广效果数据 |
| M04-02 | 推广排行榜 | P0 | 员工推广排名 |
| M04-03 | 推广明细查询 | P1 | 查看具体推广记录 |
| M04-04 | 部门统计 | P1 | 按部门统计推广数据 |

#### 3.5.3 详细功能说明

**M04-01 推广数据统计**
- **统计维度**：
  - 员工维度：每个员工的推广数据
  - 时间维度：日/周/月/自定义时间段
  - 部门维度：各部门的推广数据
- **统计指标**：
  - 扫码次数：推广码被扫描的次数
  - 新增关注：通过推广码关注公众号的人数
  - 取关人数：推广带来的用户中取关的人数
  - 净增关注：新增关注 - 取关人数
  - 关注转化率：新增关注 / 扫码次数 × 100%
- **数据更新**：
  - 实时更新：关注/取关事件触发时
  - 定时统计：每小时汇总一次

**M04-02 推广排行榜**
- **排名类型**：
  - 按新增关注排名
  - 按净增关注排名
  - 按扫码次数排名
- **展示形式**：
  - 列表排名：表格形式展示
  - 可视化排名：柱状图/排行榜样式
- **排名周期**：
  - 今日排名
  - 本周排名
  - 本月排名
  - 总排名
- **激励功能**：
  - Top 10 标识
  - 可配置奖励规则

**M04-03 推广明细查询**
- **查询条件**：
  - 员工：选择具体员工
  - 时间范围：选择时间段
  - 状态：全部/仅扫码/已关注
- **展示字段**：
  - 扫码时间
  - 用户昵称（脱敏处理）
  - 状态（仅扫码/已关注/已取关）
  - 关注时间
- **操作**：
  - 导出明细Excel

**M04-04 部门统计**
- **统计内容**：
  - 各部门推广数据汇总
  - 部门人均推广效果
  - 部门推广趋势
- **展示形式**：
  - 部门对比表格
  - 部门趋势图表

#### 3.5.4 数据模型

```sql
-- 推广统计表（按日汇总）
CREATE TABLE promotion_statistics_daily (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  employee_id BIGINT NOT NULL COMMENT '员工ID',

  stat_date DATE NOT NULL COMMENT '统计日期',

  subscribe_count INT DEFAULT 0 COMMENT '新增关注数',
```

## 七、非功能性需求

### 7.1 性能要求
- **响应时间**：页面加载时间 < 2秒
- **并发处理**：支持1000+并发用户
- **数据处理**：10万+员工数据正常查询

### 7.2 安全要求
- **数据加密**：敏感数据加密存储
- **访问控制**：严格的权限验证
- **操作审计**：关键操作日志记录
- **数据脱敏**：用户昵称等隐私信息脱敏展示

### 7.3 可用性要求
- **系统可用性**：99.9%以上
- **数据备份**：每日自动备份
- **容错机制**：微信接口调用失败重试

### 7.4 兼容性要求
- **浏览器**：Chrome、Edge、Safari最新版
- **移动端**：iOS、Android微信内置浏览器
- **微信版本**：微信 7.0+

## 十、附录

### 10.1 微信公众号API清单

| API接口 | 说明 | 用途 |
|---------|------|------|
| /cgi-bin/token | 获取access_token | 基础认证 |
| /cgi-bin/user/get | 获取关注者列表 | 同步关注状态 |
| /cgi-bin/user/info | 获取用户基本信息 | 获取用户详情 |
| /cgi-bin/qrcode/create | 创建带参数二维码 | 生成推广码 |
| /cgi-bin/message/custom/send | 发送客服消息 | 通知用户 |

### 10.2 数据字典

| 字段名 | 类型 | 说明 | 枚举值 |
|-------|------|------|-------|
| bind_status | tinyint | 绑定状态 | 0-未绑定 1-已绑定 2-已禁用 |
| follow_status | tinyint | 关注状态 | 0-未关注 1-已关注 |
| event_type | tinyint | 事件类型 | 1-关注 2-取关 |
| promo_status | tinyint | 推广记录状态 | 0-仅扫码 1-已关注 |

### 10.3 术语表

| 术语 | 说明 |
|-----|------|
| OpenID | 微信用户在公众号下的唯一标识 |
| UnionID | 微信用户在同一主体下应用的唯一标识 |
| 场景值 | 带参数二维码中的参数值，用于识别来源 |
| 场景ID | 系统内部的场景标识 |
