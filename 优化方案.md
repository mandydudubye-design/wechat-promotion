```sql
-- 统一员工表，合并 employees 和 employee_info
CREATE TABLE employees_unified (
  id INT PRIMARY KEY AUTO_INCREMENT,
  employee_id VARCHAR(50) UNIQUE NOT NULL COMMENT '工号',
  name VARCHAR(100) NOT NULL COMMENT '姓名',
  department VARCHAR(100) COMMENT '部门',
  position VARCHAR(100) COMMENT '职位',
  phone VARCHAR(20) COMMENT '手机号',

  -- 微信绑定
  openid VARCHAR(100) UNIQUE COMMENT '微信OpenID',
  bind_status TINYINT DEFAULT 0 COMMENT '0-未绑定 1-已绑定',
  bind_time DATETIME COMMENT '绑定时间',

  -- 关注状态
  follow_status TINYINT DEFAULT 0 COMMENT '0-未关注 1-已关注',
  follow_time DATETIME COMMENT '关注时间',
  unfollow_time DATETIME COMMENT '取关时间',

  -- 推广信息
  qr_code_url VARCHAR(500) COMMENT '推广二维码URL',
  scene_id VARCHAR(100) UNIQUE COMMENT '场景ID',
  total_scan_count INT DEFAULT 0,
  total_follow_count INT DEFAULT 0,

  status TINYINT DEFAULT 1 COMMENT '1-在职 0-离职',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 1.2 订阅号方案（备选）

**识别码实现**（补充功能）：
```typescript
// 1. 生成验证码（发送短信）
async function generateVerificationCode(employeeId: string, phone: string) {
  const code = Math.floor(100000 + Math.random() * 900000).toString();
  await pool.query(
    'INSERT INTO verification_codes (phone, code, expired_at) VALUES (?, ?, DATE_ADD(NOW(), INTERVAL 5 MINUTE))',
    [phone, code]
  );
  // 发送短信验证码
  await sendSms(phone, `您的绑定验证码是：${code}，5分钟内有效`);
  return code;
}

// 2. 处理员工发送的验证码
async function handleVerificationCode(openid: string, content: string) {
  const [employee] = await pool.query(
    'SELECT * FROM employees_unified WHERE employee_id = ?',
    [content.substring(0, 7)] // 假设工号是EMP001 + 6位验证码
  );

  if (employee.length > 0) {
    await pool.query(
      'UPDATE employees_unified SET openid = ?, bind_status = 1, bind_time = NOW() WHERE employee_id = ?',
      [openid, content.substring(0, 7)]
    );
  }
}
```

## 六、总结

### 方案建议

1. **服务号方案是当前最优解**，建议保持并优化
2. **订阅号识别码方案作为备选**，仅在无法使用服务号时采用
3. **统一数据表结构**，避免数据分散
4. **重点优化UI/UX**，提升用户体验

### 优先级

**P0（必须）**：
- 数据表统一
- UI优化（仪表盘、图表、动画）
- 移动端适配

**P1（重要）**：
- 实时通知
- 数据导出
- 权限管理

**P2（可选）**：
- 订阅号支持
- 高级数据分析
- 更多图表类型
