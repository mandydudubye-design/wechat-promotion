# 微信公众号测试指南

## 📋 测试前准备

### 方案一：使用微信测试号（推荐，无需认证）

微信官方提供测试号，可以直接测试所有接口功能，无需企业认证。

**优点：**
- ✅ 免费，无需认证
- ✅ 可以测试所有API接口
- ✅ 支持生成二维码、获取用户信息等
- ✅ 适合开发测试

**限制：**
- 最多100个关注者
- 只能扫码关注，无法搜索关注

### 方案二：使用正式公众号

需要企业认证的公众号，需要申请AppID和AppSecret。

**优点：**
- ✅ 无关注者数量限制
- ✅ 完整功能

**缺点：**
- ❌ 需要企业资质
- ❌ 需要微信认证

## 🚀 方案一：使用微信测试号（推荐）

### 1. 获取测试号信息

1. 访问微信公众平台测试号页面：
   https://mp.weixin.qq.com/debug/cgi-bin/sandbox?t=sandbox/login

2. 使用微信扫码登录

3. 登录后会获得：
   - **appID**: 测试号appID
   - **appsecret**: 测试号密钥

### 2. 配置服务器

打开项目 `.env` 文件，更新微信配置：

```env
# 微信公众号配置（使用测试号）
WECHAT_APP_ID=wx1234567890abcdef  # 替换为你的测试号appID
WECHAT_APP_SECRET=your_app_secret_here  # 替换为你的测试号appsecret
WECHAT_TOKEN=wechat_promotion_token
WECHAT_ENCODING_AES_KEY=
```

### 3. 配置服务器URL

在测试号管理页面配置服务器地址：

- **URL**: `http://你的域名或IP:3000/api/wechat/callback`
  - 如果是本地测试，需要使用内网穿透工具（如ngrok、花生壳等）
  
- **Token**: `wechat_promotion_token`（与.env中的WECHAT_TOKEN一致）

- **EncodingAESKey**: 随机生成即可（可以不填）

### 4. 配置内网穿透（本地测试）

如果是本地开发，需要配置内网穿透将本地服务暴露到公网：

#### 选项A：使用ngrok（推荐）

1. 下载ngrok: https://ngrok.com/download
2. 注册账号获取authtoken
3. 运行ngrok:
```bash
ngrok http 3000
```

4. 会获得一个公网地址，例如：`https://abc123.ngrok.io`
5. 在微信测试号配置URL：`https://abc123.ngrok.io/api/wechat/callback`

#### 选项B：使用花生壳

1. 下载花生壳客户端
2. 配置内网映射到本地3000端口
3. 使用花生壳提供的域名

#### 选项C：部署到云服务器

如果有云服务器，可以直接部署：
- 阿里云、腾讯云等
- 确保服务器安全组开放3000端口

## 📱 测试步骤

### 步骤1：启动服务器

```bash
cd backend
npm run dev
```

服务器会在 `http://localhost:3000` 启动

### 步骤2：验证服务器配置

在微信测试号页面点击"提交"按钮，验证服务器配置是否成功。

如果验证成功，页面会显示：
- 服务器配置 **已启用**
- 接口配置 **成功**

### 步骤3：获取测试号二维码

在测试号页面，会显示测试号的二维码：
- 使用微信扫码关注测试号
- 关注后可以在测试号页面看到关注者列表

### 步骤4：测试API接口

#### 4.1 测试登录

```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d "{\"username\":\"admin\",\"password\":\"admin123\"}"
```

#### 4.2 测试创建推广（生成二维码）

```bash
curl -X POST http://localhost:3000/api/promotion/create \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d "{\"employee_id\":\"EMP001\",\"account_id\":1,\"description\":\"测试推广\"}"
```

返回会包含二维码URL。

#### 4.3 扫码测试

1. 使用微信扫描生成的二维码
2. 会触发关注事件
3. 后端会记录关注信息
4. 可以通过API查询关注记录

### 步骤5：测试其他功能

- 查询推广记录
- 更新推广数据
- 查询关注记录
- 获取统计数据
- 导出数据

## 🐛 常见问题

### 问题1：服务器配置验证失败

**可能原因：**
- Token不匹配
- 服务器地址无法访问
- 端口未开放
- 内网穿透配置错误

**解决方案：**
1. 检查.env中的WECHAT_TOKEN是否与微信配置一致
2. 确保服务器地址可以通过公网访问
3. 检查防火墙/安全组是否开放3000端口
4. 检查内网穿透是否正常工作

### 问题2：二维码生成失败

**可能原因：**
- AppID或AppSecret错误
- access_token获取失败
- 网络问题

**解决方案：**
1. 检查.env中的微信配置是否正确
2. 查看后端日志，确认错误信息
3. 检查网络连接

### 问题3：扫码无反应

**可能原因：**
- 场景值格式错误
- 公众号未正确配置
- 二维码已过期

**解决方案：**
1. 检查场景值格式是否正确
2. 确认服务器配置已启用
3. 重新生成二维码

### 问题4：获取不到用户信息

**可能原因：**
- 用户未关注
- openid错误
- API调用失败

**解决方案：**
1. 确认用户已关注公众号
2. 检查openid是否正确
3. 查看日志确认错误
